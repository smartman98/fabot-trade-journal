# PRD — FABOT 매매 일지

## 문제 정의
F&G(Fear & Greed) 지수 기반 매매 신호는 대시보드로 확인할 수 있지만, 실제로 그 신호를 보고 언제·얼마에·어떤 종목을 매수/매도했는지 기록하고 되돌아볼 곳이 없다. 시간이 지나면 "그때 왜 그렇게 매매했는지", "신호대로 실행했는지"를 기억하기 어렵다.

## 타겟 유저
F&G 기반 규칙 투자를 하는 본인(1인용). 로그인 없이 개인 기록용으로 사용한다.

## 핵심 기능 (MVP)
1. 매매 기록 추가 — 날짜, 종목, 매수/매도 구분, 수량, 가격, 그 시점 F&G 점수, 메모
2. 매매 기록 목록 조회 — 최신순 정렬, 매수/매도 구분 표시
3. 매매 기록 상세 조회 — 기록 하나를 클릭하면 전체 내용 확인
4. 매매 기록 수정 · 삭제
5. 요약 통계 — 총 매수 횟수, 총 매도 횟수, 총 매수금액, 총 매도금액

## 화면 구성
1. **목록 화면**: 매매 기록 리스트(카드 형태) + 상단 요약 통계 + "기록 추가" 버튼
2. **입력/수정 폼 화면**: 새 기록 추가 또는 기존 기록 수정
3. **상세 화면**: 기록 하나 클릭 시 전체 내용(모달 또는 별도 영역)

## 데이터 구조
```
trades
- id            bigint, PK
- trade_date    date            (매매 날짜)
- ticker        text            (종목명, 예: TQQQ)
- action        text            ('buy' | 'sell')
- quantity      numeric         (수량)
- price         numeric         (매매 단가)
- fg_score      numeric, null   (그 시점 F&G 점수)
- memo          text, null      (메모)
- created_at    timestamptz     (등록 시각)
```

## 기술 구조
- 프론트엔드: 정적 HTML/CSS/JS (fetch로 백엔드 API 호출)
- 백엔드: Node.js + Express (CRUD API 서버)
- DB: Supabase Postgres (백엔드에서만 접근, 프론트는 직접 접근하지 않음)
- 배포: Render (백엔드 + 정적 프론트 통합 서빙)
